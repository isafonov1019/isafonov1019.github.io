# Сайт православного храма

Статический сайт храма: расписание богослужений, новости, история, контакты
и пожертвования. Настоятель обновляет контент сам — через админку, без
программирования.

## Как устроено

| Часть | Технология |
|---|---|
| Сайт | [Astro](https://astro.build) — собирает быстрые статические страницы |
| Контент | Markdown-файлы в `src/content/` (схемы — `src/content.config.ts`) |
| Админка | Decap CMS на `/admin`, русский интерфейс (`public/admin/config.yml`) |
| Стили | Чистый CSS: `src/styles/global.css`; шрифты PT Serif / PT Sans (Fontsource) |
| Хостинг | Netlify (`netlify.toml`) |

Общие данные храма (название, адрес, телефон, реквизиты) лежат в одном файле
`src/content/settings/site.md` и подставляются по всему сайту.

## Команды

```bash
npm install        # установить зависимости (один раз)
npm run dev        # сайт локально: http://localhost:4321
npm run cms        # помощник админки (нужен для /admin локально)
npm run build      # боевая сборка в папку dist/
```

## Как редактировать контент

**Через админку (основной путь):** запустить `npm run dev` и `npm run cms`
(два терминала), открыть http://localhost:4321/admin, нажать «Войти».
После публикации на Netlify — вход через GitHub на `сайт/admin`.

**Файлами:** контент — обычный Markdown в `src/content/`
(services — расписание, news — новости, clergy — духовенство,
gallery — галерея, pages — тексты, settings — данные храма).

## Структура

```
src/
├── content/          # весь редактируемый контент
├── components/       # шапка, подвал, эмблема-крест
├── layouts/          # общий каркас страниц
├── pages/            # страницы сайта (адрес = имя файла)
├── styles/global.css # дизайн-система
└── data/             # общие данные (пункты меню)
public/
└── admin/            # админка (Decap CMS, self-hosted)
```

Пометки вида `[УТОЧНИТЬ: …]` в контенте — заглушки: реальные данные храма
нужно получить у настоятеля и внести через админку.
